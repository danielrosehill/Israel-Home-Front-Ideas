# רעיונות לעורף הישראלי

אוסף קטן של רעיונות שרשמתי לשיפור ההתרעה בעורף בישראל (Pikud HaOref / פיקוד העורף), שנצברו במהלך שתי המלחמות עם איראן.

> הצעות אלו מוגשות בענווה ועם כבוד עמוק לאנשי פיקוד העורף וכוחות החירום ששמרו עלינו בטוחים. איני איש מקצוע בתחום הביטחון — אלא תושב שבילה זמן רב בממ"ד, וחשב על שיפורי UX ותקשורת קטנים שעשויים לעזור לאחרים. הרעיונות משותפים בתום לב, ללא כל יומרה למומחיות, ובתקווה שאפילו אחד מהם יוכל להיות שימושי.

**[הורד את האוסף המלא כקובץ PDF](build/Israel-Home-Front-Ideas.pdf)**

## מבנה מאגר זה

הרעיונות מקובצים לפי נושאים. כל רעיון עוקב אחר תבנית פשוטה:

- **הבעיה** — מה ששמתי לב שלא עבד היטב, או שיכול היה להיות טוב יותר.
- **פתרון מוצע** — סקיצה כללית של מה שעשוי לעזור.

## אינדקס רעיונות

### 1. האפליקציה הרשמית (אפליקציית פיקוד העורף)
_רעיונות הקשורים לאפליקציית ההתרעה הרשמית — התראות, חווית משתמש (UX), נגישות, אמינות._

### 2. צופרים ומסירת התרעות
_רעיונות לגבי אופן הגעת ההתרעות לאנשים: צופרים, התראות דחיפה (push notifications), הודעות SMS, שידור סלולרי (cell broadcast), ערוצי גיבוי._

- [שילוב התרעות פיקוד העורף עם מערכות SCADA של רמזורים](ideas/siren-alert-delivery/traffic-light-scada-integration.md) — פינוי כבישים ומתן חיווי ויזואלי מובנה לנהגים במהלך התרעות.

### 3. מידע במהלך אירוע ולאחריו
_מה שאנשים צריכים לדעת ברגע שהתרעה מופעלת: אותות רגיעה, מידע על פגיעה, הנחיות._

- [מצב רגיעה מובחן כאשר התרעה מוקדמת אינה מלווה באזעקת אמת](ideas/info-during-after/early-warning-without-alert-state.md) — סגירת מעגל העמימות עם וריאנט הודעה משלו.
- [הוספת שדה "הנחיה עדכנית" לפיד נתוני ההתרעה](ideas/info-during-after/current-guidance-field-in-feed.md) — הפיכת ההנחיה הקבועה למפורשת במקום משתמעת.

### 4. נגישות והכלה
_הפיכת ההתרעות לנגישות לכולם: עולים, תיירים, אנשים עם מוגבלויות, קשישים, דוברי שפות שאינן עברית._

### 5. מקלטים ולוגיסטיקת ממ"דים
_רעיונות סביב מקלטים ציבוריים, מוכנות ממ"דים, ומידע על המרחב המוגן הקרוב ביותר._

רעיונות המקלטים יוצרים ערימה שכבתית — מוסד ← נתונים ← אפליקציה — ומומלץ לקרוא אותם יחד:

- [רשות מקלטים ציבוריים](ideas/shelters-mamad/public-shelter-authority.md) — *שכבת מוסד.* גוף לאומי, רישום GIS, קוד תקינות, פיקוח ואכיפה.
- [פורמט רישום סטנדרטי וניווט פיזי](ideas/shelters-mamad/standard-listing-format-and-wayfinding.md) — *שכבת נתונים.* רשימות קריאות מכונה, קואורדינטות מדויקות, נתיבי גישה מתועדים.
- [אפליקציית איתור מקלטים עירונית חובה](ideas/shelters-mamad/municipal-shelter-finder-app.md) — *שכבת צרכן.* מבוססת מיקום גיאוגרפי, נגישה, תמונות/סרטונים, מקור אמת יחיד.
- [יתירות תקשורת כחובת בסיס בכל מקלט ציבורי](ideas/shelters-mamad/comms-redundancy-in-shelters.md) — סלולר, Wi-Fi, טאבלט WEA מוקשח, ורדיו AM/FM בכל מקלט.
- [שירותים לשהייה ממושכת — פחות מקלטים אך מאובזרים יותר](ideas/shelters-mamad/long-stay-amenities.md) — מזרנים, אטמי אוזניים, מיזוג אוויר, והפשרה המפורשת.

### 6. נתונים, ממשקי API ומערכת אקולוגית למפתחים
_התשתית סביב נתוני התרעה: פידים ציבוריים, ממשקי API, שרתי MCP, וקהילת האפליקציות התלויות בהם._

- [מתן ממשק API ציבורי ומתועד](ideas/data-apis/public-documented-api.md) — החלפת פיזור הסקרייפרים במקור אמת מנוהל באופן מקצועי.
- [מידול פורמלי של סכימת ה-payload של ההתרעה מנתונים שנלכדו בפועל](ideas/data-apis/formal-payload-model.md) — גזירת JSON Schema קנונית מתעבורה נצפית.
- [סכימת payload נצפית (v0)](ideas/data-apis/observed-payload-schema.md) — מודל אמפירי מלכידה חיה, המניע את מאמץ המידול הפורמלי.
- [שמות אזורים רשמיים רב-לשוניים + מזהי אזורים יציבים](ideas/data-apis/official-multilingual-area-names.md) — ביטול טבלאות התרגום לכל אפליקציה שנשברות עבור דוברי שפות שאינן עברית.
