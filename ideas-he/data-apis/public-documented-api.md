# ספקו API ציבורי ומתועד

**מקטע:** נתונים, ממשקי API ומערכת אקולוגית למפתחים
**תועד בתאריך:** 2026-04-06

## בעיה

קיימים כיום למעלה מ-100 סקריפרים של אזעקת צבע אדום הפועלים באופן עצמאי. ניתן להניח באופן סביר שמערכת ההתרעה הציבורית של ישראל נוטרה באופן פעיל על ידי גורמים עוינים עוד לפני המלחמות האחרונות, ובוודאי שהיא מנוטרת כעת – כך שהטיעון הרגיל של "אבטחה באמצעות עמימות" למניעת פיד ציבורי הוא חלש.

בהיעדר נימוק ברור של אבטחת מידע שיצדיק את ההפך, המצב הנוכחי – התפשטות של פרויקטים קהילתיים המסתמכים כולם על גירוד (scraping) של נקודות קצה לא מתועדות – הוא שביר, מועד לשגיאות וקשה לאמינות. סקריפרים נשברים בשקט, יוצרים כפילות מאמץ ומפיקים נתונים לא עקביים בדיוק ברגעים שבהם האמינות חשובה ביותר.

## פתרון מוצע

API זמין לציבור, מנוהל באופן מקצועי ומתועד מפיקוד העורף, המספק מקור אמת יחיד לנתוני התרעות. באופן אידיאלי, זה יכלול:

- API יציב, מבוסס גרסאות, בפורמט HTTP/JSON, עם סכימות ברורות ומפרט OpenAPI.
- שרת MCP נלווה כך שסוכני AI ועוזרים יוכלו לצרוך נתוני התרעות באמצעות ממשק סטנדרטי.
- הסכמי רמת שירות (SLA), מגבלות קצב (rate limits) ודף סטטוס מפורסמים.
- תנאי שימוש ברורים המאפשרים שימוש באפליקציות, לוחות מחוונים (דאשבורדים) ומחקרים במורד הזרם.
- נקודות קצה לנתונים היסטוריים לצורך ניתוח לאחר אירוע.

המטרה אינה להחליף את האפליקציה הרשמית, אלא להעניק למערכת האקולוגית סביבה בסיס איתן, להפחית את הנטל על סקריפרים מתנדבים ולשפר את האמינות עבור כל מי שתלוי בנתונים.

## הערות

- כדאי לבדוק האם קיים פיד חצי-רשמי כלשהו כיום, ומה מצב התיעוד שלו.
- שרת MCP בפרט יהווה מודרניזציה קטנה אך משמעותית – הוא יאפשר לעוזרים לענות על השאלה "האם יש התרעה פעילה ב-X?" מבלי שכל מפתח יצטרך להמציא את הגלגל מחדש.
- קשור קשר הדוק ל-[הוספת שדה "הנחיות עדכניות" לפיד נתוני ההתרעות](../info-during-after/current-guidance-field-in-feed.md). שדה ה-"current-guidance" הוא סוג של שיפור סכימה ש-API ציבורי מתועד ומבוסס גרסאות יאפשר לשלוח בקלות – ולהיפך, API ציבורי ללא שדה מצב חד-משמעי פשוט יפיץ את העמימות הקיימת כיום ליותר צרכנים.
