# סכימת מטען נצפית (v0) — מודל אמפירי מלכידה חיה

**מקטע:** נתונים, ממשקי API ומערכת אקולוגית למפתחים
**נלכד בתאריך:** 2026-04-06

זהו שרטוט עבודה אמפירי של מטען ההתרעה החיה כפי שנצפה מריליי פועל מול הפיד המקורי של פיקוד העורף. זו **אינה** מפרט רשמי — זה מה שה-wire באמת נראה כמוהו. הוא קיים כדי לתת לרעיון [למדול פורמלית את סכימת מטען ההתרעה מנתונים שנלכדו בפועל](formal-payload-model.md) נקודת התחלה קונקרטית.

## נקודות קצה שנצפו

שתי צורות של נקודות קצה נחשפות על ידי הפיד המקורי (כפי שמועבר מקומית):

- **התרעות חיות** — מה שמשודר כרגע.
- **היסטוריה** — רשימה מתגלגלת של שורות התרעה אחרונות.

## מטען התרעות חיות

```jsonc
{
  "alerts": [],            // מערך; ריק כשאין כלום פעיל
  "timestamp": 1775483085.268129  // שניות יוניקס אפוק (מספר עשרוני)
}
```

כאשר במצב סרק, `alerts` הוא מערך ריק. ה-`timestamp` הוא יוניקס אפוק בצד השרת — שימו לב שהוא **אינו** נישא בתוך אובייקטי ההתרעה עצמם.

## מטען היסטוריה

```jsonc
{
  "history": [
    {
      "alertDate": "2026-04-06 16:43:54",  // זמן מקומי, "YYYY-MM-DD HH:MM:SS", ללא אזור זמן
      "title":     "ירי רקטות וטילים",     // תווית עברית לסוג ההתרעה
      "data":      "גדות",                  // שם אזור מושפע יחיד (עברית)
      "category":  1                         // קוד סוג מספרי (ראו להלן)
    }
  ]
}
```

### הערות שדה

- **`alertDate`** הוא מחרוזת זמן מקומי **ללא מזהה אזור זמן**. צרכנים חייבים להניח זמן מקומי בישראל (IST/IDT) ולטפל בשעון קיץ בעצמם. זוהי חולשה אמיתית של הסכימה.
- **`title`** הוא תווית עברית קריאה לבני אדם. אין שדה מקביל באנגלית, אין תג שפה, ואין מזהה יציב קריא למכונה — `category` הוא המפתח האמין היחיד.
- **`data`** הוא שם אזור יחיד כמחרוזת עברית. אין **קואורדינטות, אין פוליגון, אין מזהה אזור**, ואין קישור למערך הנתונים הקנוני של פוליגונים שאפליקציות צרכניות מתיישרות מולו בסופו של דבר.
- **`category`** הוא enum של מספר שלם קטן; ראו את הרשימה החלקית להלן.
- **שורה אחת לכל אזור מושפע.** אירוע יחיד המכסה 30 אזורים מופיע כ-30 שורות נפרדות, עם אותו `alertDate` ו-`title` וערכי `data` שונים. אין מזהה אירוע המקבץ שורות אלו לאירוע יחיד.

## קטגוריות שנצפו

מחלון לכידה קצר יחיד — רשימה זו אינה שלמה ויש להרחיב אותה ככל שייראו וריאנטים נוספים:

| `category` | `title` (Hebrew)         | משמעות                           |
|-----------:|--------------------------|----------------------------------|
| 1          | ירי רקטות וטילים         | ירי רקטות וטילים                 |
| 2          | חדירת כלי טיס עוין       | חדירת כלי טיס עוין               |
| 13         | האירוע הסתיים             | האירוע הסתיים (הכל בסדר)         |

## מה זה מאשר לגבי רעיונות אחרים במאגר זה

הסכימה האמפירית מאמתת כמה מהבעיות שהועלו במקומות אחרים באוסף:

- **אין שדה "הנחיות עדכניות".** מאשר [הוספת שדה "הנחיות עדכניות" לפיד נתוני ההתרעות](../info-during-after/current-guidance-field-in-feed.md). בין שורת קטגוריה-1 לשורת קטגוריה-13, אין אות בתוך המטען שאומר "הישארו במרחב מוגן". צרכנים חייבים לשחזר זאת.
- **"הכל בסדר" הוא אירוע לכל אזור, לא מצב.** אותה בעיה מזווית אחרת — קטגוריה 13 מופעלת פעם אחת לכל אזור, ואז הפיד שקט. צרכנים המתחברים מחדש אינם יכולים לדעת אם הם באמצע אירוע או אחריו.
- **אין קוד "רגיעה" מובחן.** מאשר [מצב רגיעה מובחן כאשר התרעה מוקדמת אינה מלווה באזעקת אמת](../info-during-after/early-warning-without-alert-state.md). קטגוריה 13 היא המסיים היחיד שנראה עד כה.
- **אין קואורדינטות.** מאשר [פורמט רישום סטנדרטי וניווט פיזי](../shelters-mamad/standard-listing-format-and-wayfinding.md) בקצה פיד הנתונים של השרשרת — אפילו פיד ההתרעות משאיר את הרזולוציה הגיאוגרפית כתרגיל לצרכן.
- **אין אזור זמן, אין מזהה אירוע, אין תג שפה.** כל אחד מהם הוא דבר קטן בפני עצמו, אך יחד הם מסבירים מדוע כל סקרייפר מיישם מחדש באופן עצמאי את אותה קומץ היוריסטיקות שבריריות.

## צעדים הבאים לחיזוק זה למודל אמיתי

- ללכוד ברציפות (קולט פסיבי ארוך טווח) כדי למנות כל קוד קטגוריה שמופיע בפועל, כולל הנדירים.
- להצליב מחרוזות `data` שנצפו מול מערך הנתונים הקנוני של פוליגוני פיקוד העורף כדי לאשר יציבות שמות האזורים.
- לגזור סכימת JSON מהקורפוס שנלכד ולפרסם אותה עם מטענים לדוגמה.
- לבנות כלי CLI קטן לאימות כך שכל מפתח יוכל להעביר דרכו מטען שנלכד.
